## 답변

웹팩은 모듈 번들러로, 여러 파일들을 하나로 묶어주는 역할을 하며, 바벨은 JavaScript 컴파일러로 최신 문법을 이전 버전의 JavaScript로 변환해주는 역할을 합니다. <br/>
웹팩은 프로젝트의 모든 자원(JS, CSS, 이미지 등)을 관리하고 최적화하며, 바벨은 코드의 호환성을 보장하는 데 중점을 둡니다.

### Q. 웹팩의 주요 기능과 장점은 무엇인가요?

웹팩은 다음과 같은 주요 기능을 제공합니다:

1. **모듈 번들링**

   - 여러 파일을 하나로 묶어 HTTP 요청을 최소화
   - CommonJS, AMD, ES6 모듈 등 다양한 모듈 시스템 지원
   - 의존성 트리를 자동으로 분석하고 관리

2. **로더(Loader)**

   - JavaScript, CSS, 이미지 등 다양한 파일 타입별로 적절한 변환 수행

3. **플러그인(Plugin)**

   - 번들 최적화, 환경 변수 주입, HTML 생성 등 번들링 과정에서 추가 작업 수행

4. **코드 스플리팅**
   - 필요한 코드만 로드하여 초기 로딩 시간 단축
   - 동적 임포트를 통한 지연 로딩 지원
   - 공통 모듈 분리를 통한 캐시 효율성 향상

### Q. 바벨의 주요 기능에 대해 설명해주세요.

바벨은 다음과 같은 주요 기능을 제공합니다:

1. **트랜스파일링**

   - ES6+ 문법을 ES5로 변환
   - JSX를 일반 JavaScript로 변환
   - TypeScript를 JavaScript로 변환

2. **폴리필**

   - 새로운 JavaScript 기능을 이전 버전에서도 사용 가능하게 함
   - core-js를 통한 기능 구현
   - regenerator-runtime을 통한 async/await 지원

3. **플러그인과 프리셋**
   - 개별 문법 변환을 위한 플러그인
   - 여러 플러그인을 모아놓은 프리셋
   - 커스텀 변환 규칙 정의 가능
