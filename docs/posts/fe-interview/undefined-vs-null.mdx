## 답변

모두 값이 없음을 나타내지만 `undefined`는 변수가 선언되었지만 값이 할당되지 않았음을 나타내는 타입,
`null`은 어떤 값이 의도적으로 비어있음을 표현하는 타입입니다.

- `undefined`는 자바스크립트 엔진이 변수를 초기화할 때 사용하고, `null`은 개발자가 의도적으로 할당합니다.

  ```js
  let u;
  console.log(u); // undefined (값을 할당하지 않음)

  let n = null;
  console.log(n); // null (개발자가 null을 할당)
  ```

- 둘을 동등 연산자(`==`)로 비교하면 참이지만, 일치 연산자(`===`)로 비교하면 거짓입니다. 동등 연산자는 자료형 변환을 수행하기 때문입니다.

  ```js
  typeof undefined; // 'undefined'
  typeof null; // 'object'

  undefined === null; // false
  undefined == null; // true
  ```
